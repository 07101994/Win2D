Pixel formats

The DirectXPixelFormat enum includes all the many and varied pixel formats used by Direct3D and DXGI, but only a few of these options are supported by Win2D (or by Direct2D upon which Win2D is built).

If in doubt, pixel format B8G8R8A8UIntNormalized and CanvasAlphaMode Premultiplied are good defaults for most purposes.

All the formats listed below are supported by Direct3D feature level 11 GPU hardware (used in desktop computers and most tablets). Feature level 9 GPUs (which are found in phones) only support a subset. If you want to use one of the formats marked as Not supported on all devices, you should catch exceptions if the resource creation fails and be prepared to fall back on one of the universally available options.

1 The block compressed formats (BC1, BC2, and BC3) are not yet fully supported by Win2D. It is possible to create block compressed bitmaps using CanvasBitmap.CreateFromBytes, to draw using them, and to interop them from native Direct2D or Direct3D, but Win2D does not provide any way to load images of these formats, and methods such as GetPixelBytes, SetPixelBytes, and CopyPixelsFromBitmap do not work correctly on block compressed data. We plan to improve this in a future version of Win2D.
